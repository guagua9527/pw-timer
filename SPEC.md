# 陪玩计时计费工具 — SPEC

## 1. 概念与愿景

专为陪玩场景设计的计时计费工具。核心是「开始/结束」单按钮计时流，配合手填表格。界面温暖柔和，NaiveUI 组件库驱动，奶油色系定制主题，柔和不刺眼。

## 2. 设计语言

- **组件库**: NaiveUI (Vue3)
- **色彩 (CSS Variables)**:
  - 主色: `#7C6F64` (暖灰棕)
  - 强调: `#E8985E` (琥珀橙)
  - 背景: `#FAF7F2`
  - 卡片: `#FFFFFF`
  - 文字: `#3D3D3D`
  - 次要: `#8A8A8A`
  - 边框: `#EAE0D5`
  - 成功: `#7CAE7A`
  - 危险: `#D96E6E`
- **字体**: "Noto Sans SC", "DM Sans"
- **阴影**: `0 2px 12px rgba(0,0,0,0.06)`
- **动效**: 过渡 200ms ease-out

- 移动端: 边栏叠到下方

## 4. 功能细节

### 4.1 计时按钮
- **未计时**: 琥珀色 `"#E8985E"`，显示「开始计时」
  - 点击 → 新增一行，开始时间=now，单价=默认单价，按钮变绿
- **计时中**: 绿色 `"#7CAE7A"`，显示「结束计时」
  - 点击 → 同一行写入结束时间，按钮恢复

### 4.2 表格 (n-data-table)
- 列: 开始时间 | 结束时间 | 单价(¥/h) | 时长 | 总价(¥) | 操作
- **所有单元格内容居中**
- **开始/结束时间**: `NDatePicker type="datetimerange"`，format='HH:mm:ss'，每列独立选择，可点选或手动输入
- **单价**: `NInputNumber`，默认值为默认单价，可单独修改
- **时长**: 自动计算，`X小时Y分`，结束<开始自动+24h（跨日）
- **总价**: `时长(h) * 单价`，保留2位小数
- **删除**: `NButton type="error"` 红色按钮
- 底部合计行: 灰底，加粗，显示总时长和总价

### 4.3 右侧边栏
- **默认单价**: `NInputNumber`，可填，默认 0，新记录以此为默认值
- **存单余额**: `NInputNumber`，可为负数
- **消费金额**: 只读 = 表格总价合计（向下取整）
- **剩余金额**: `余额 - 消费`，负数红色显示

### 4.4 操作按钮
- **💸 扣费**: 余额 = 余额 - 消费金额，清空表格，余额可为负
- **🗑 清空**: 清空表格；勾选「清空余额」则同时将余额归零

### 4.5 数据持久化
- key: `playtime_records`
- 结构: `{ records: [{start, end, price}], balance: number, defaultPrice: number }`
- `watch` 深度监听 → 自动保存 localStorage

## 5. 组件清单

| 组件 | NaiveUI | 定制 |
|------|---------|------|
| 按钮 | `n-button` | 颜色主题变量 |
| 表格 | `n-data-table` | 居中样式 |
| 输入 | `n-input`, `n-input-number` | 时间选择框 |
| 复选框 | `n-checkbox` | 清空余额勾选 |
| 卡片 | `n-card` | 圆角 12px |
| 布局 | `n-grid`, `n-gi` | 响应式 |

## 6. 技术方案

- Vue 3 + Composition API + `<script setup>`
- Vite
- NaiveUI (主题定制 via CSS variables + `n-config-provider`)
- 原生 CSS，NaiveUI 覆盖样式
- `watch` 深度监听自动持久化
